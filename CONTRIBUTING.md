# Contributing

Thanks for helping make cloud computing easier for non‑IT learners! 🙌

This repo is the open curriculum for **Demystified Cloud with Ammar Mudiraaj**. Contributions that keep it beginner‑first are very welcome.

## Ways to help

- **Fix errors** — a wrong service name, outdated info, broken link, or typo.
- **Clearer analogies** — suggest a simpler everyday metaphor for a tricky concept.
- **Translations** — help localize titles, summaries, or outlines.
- **New/updated outlines** — improve a video outline following the template.

## Ground rules (the method)

Keep every contribution aligned with the teaching principles:

1. **Non‑IT first** — explain with an analogy before any technical term.
2. **Define before use** — never use a term the curriculum hasn't taught yet.
3. **Provider order** — always list as **AWS → Azure → GCP → OCI → Alibaba**.
4. **Portable framing** — "same idea, different name/button," not vendor loyalty.
5. **Cost‑safety** — any hands‑on step must note free‑tier use and what to shut down.

## How to submit

1. **Open an issue** describing the fix/idea (use the video ID, e.g., `P1-NET-F`).
2. For text changes, **fork → edit the Markdown → open a pull request**.
3. Keep PRs focused (one topic per PR) and reference the issue.

## Style

- Short sentences. Plain words. One big analogy per concept.
- Use the existing ID scheme and Markdown structure.
- Cite the official docs when updating a service name or limit.

## Reporting a service‑name change

Cloud providers rename things. If you spot one, open an issue titled `naming: <old> → <new>` with a link to the official source, and we'll update the curriculum and note the change.
